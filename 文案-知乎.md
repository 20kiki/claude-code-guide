# Claude Code 避坑指南：这些命令能让你少走一半弯路

用 Claude Code 写代码一段时间后，每个人都会遇到同样的问题：**AI 改了什么代码自己完全没看、上下文塞满无效对话导致回答越来越蠢、一个任务纠正好几次后整个会话直接废掉。**

这个项目把日常使用中最实用的命令和技巧整理成了一站式速查手册，重点讲了三个官方文档没太说明白但实际最重要的操作：

---

### 1. 上下文管理三件套 — 避免「越聊越傻」

这是项目最大的亮点。Claude Code 在超长对话中，你的每一次纠正、临时提问都会永久写入会话历史，挤占 token 空间，30% 的上下文可能被无效内容吃掉。

- **`/rewind`**（或双击 Esc Esc）：方案跑偏了，不纠正，直接回退到出错之前。支持三种粒度：仅回退代码 / 仅回退对话 / 两者同时回退
- **`/fork`**：实现到一半想到新思路，分叉一个新会话去验证。行就用更好的指令重做，不行就切回来继续
- **`/btw`**：临时想问个问题但不想污染上下文，问答只展示一次，绝不写入对话历史

### 2. 命令速查 + 快捷键大全

按类别整理了所有斜杠命令（基础对话 / 工作模式 / 代码审查 / Git 操作 / 自动化），以及常用键盘快捷键。`@` 引用文件、`!` 执行命令、`#` 快速写入记忆这些操作也都覆盖到了。

### 3. 中文配置指南

全局默认中文回答的配置方法、终端状态栏实时显示上下文占用百分比（方便在 50% 时及时 compact），直接复制粘贴到 `settings.json` 就行。

### 4. 高效工作流模板

功能开发流、调试流、大规模重构流都有成型的 SOP，还有并行 worktree 多任务开发和后台 `/loop` 自动化巡检的实践。

### 5. 模型选择 + 省钱策略

Sonnet 日常干活、Opus 攻克复杂架构、Haiku 处理简单任务 — 最高性价比的用法说清楚了。

---

**三步上手：**

```bash
# 1. 先全局配置中文回答
#    编辑 ~/.claude/CLAUDE.md，加入语言偏好

# 2. 新项目第一步
claude
/init

# 3. 每个新任务前清空上下文
/clear
```

项目地址：https://github.com/20kiki/claude-code-guide

如果你已经在用 Claude Code 但还没试过 `/rewind` 和 `/fork`，建议先收藏再试 — 这两个命令能省掉大量不必要的纠正对话。
